PartialRetryStep
Resume path — on relaunch, a persisted session routes by outcome: real failures → Almost there (retry); nothing failed (interrupted mid-confirm, or timed-out pending) → Still processing. A running snapshot is written before the first batch confirms, so even a kill during batch 1 stays resumable. Counts (moved / failed) are surfaced from the session — never a bare 0 / 0.

Non-destructive by construction

Destination is the user’s own wallet. Source keeps its keys. Only MIGRATABLE_MINTS move; unresolvable mints are dropped, never shipped as a corrupted raw amount.

pending ≠ failed

summarizeBatch splits pending from terminal-failed. A slow confirmation is “still processing,” never a retry screen — no phantom failures.

Amount safety

Integer-string math only, no float. uiToRaw("5.7", 0) → "5" — the fractional part is truncated, not concatenated, on the funds-moving path.

Open everyone-first

No eligibility gate. Email OTP → auto-created embedded wallet, or connect an external self-custody wallet. Fees presented as Free.
